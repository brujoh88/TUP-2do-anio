## 3. Diferencia entre HTML, CSS y JavaScript

Del ejercicio de matching (p.5):

| Tecnologia | Funcion | Descripcion |
|-----------|---------|-----------|
| **HTML** | Create the structure | Controla el layout del contenido, provee estructura para el diseño, es el bloque fundamental de cualquier pagina web |
| **CSS** | Stylize the website | Aplica estilos a los elementos, adapta a diferentes pantallas (responsive), maneja el "look and feel" |

## 4. Siete puntos para evaluar una pagina web (p.6)

1. **Design** → ¿El diseño es atractivo y profesional?
2. **Navigation** → ¿Es facil moverse por el sitio?
3. **Easy of use** → ¿Es facil de usar?
4. **Accuracy** → ¿La informacion es correcta y precisa?
5. **Up to date** → ¿El contenido esta actualizado?
6. **Helpful graphics** → ¿Las imagenes y graficos ayudan?
7. **Compatibility** → ¿Funciona en distintos navegadores y dispositivos?

## 5. Texto clave: "Building Your First Website" (p.4)

- Para construir un sitio, lo primero es **HTML** (da la estructura: texto, imagenes, secciones)
- Despues se usa **CSS** (Cascading Style Sheets) para cambiar el **layout**. Controla colores, fuentes y diseño general
- CSS es un lenguaje simple, pensado para simplificar la presentacion de paginas web
- CSS permite aplicar estilos **independientemente** del HTML
- HTML usa **tags** (etiquetas), CSS usa **rule sets** (conjuntos de reglas)
- Es importante que el sitio sea **responsive** (se adapte a celulares, tablets, laptops)
- La **navegacion** debe ser clara: menus con links y botones
- Cada sitio tiene un **domain name** (ej: "google.com")

## 6. Texto clave: "Promote your Business" (p.6)

Descripcion de una web page para un small business:
- Usa **HTML para estructura** y **CSS para diseño**
- Layout **simple**: header arriba, contenido en el centro → facil de entender
- Color de fondo **gris claro** (light gray) → el texto se lee facil, es **simple and modern**
- Fuente **Arial** → simple y moderna
- Botones **azules** (blue) → color calmo, facil de ver. Texto en **blanco** para crear **contraste**
- La pagina es **responsive** → funciona bien en celulares
- CSS controla colores, fuentes y espaciado → el sitio se ve **clean and professional**

## 7. Adjetivos Comparativos y Superlativos (p.16)

Se usan para **comparar** elementos de diseño web y **justificar decisiones**.

### Reglas para formar comparativos

| Tipo | Comparativo | Superlativo | Ejemplo |
|------|-----------|------------|---------|
| **Cortos** (1 silaba) | adj + **-er** + than | the + adj + **-est** | big → bigg**er** than / the bigg**est** |
| **Largos** (2+ silabas) | **more/less** + adj + than | **the most/least** + adj | expensive → **more** expensive than |
| **Irregulares** | forma especial | forma especial | good → **better** / **the best** |

### Adjetivos del modulo con sus formas

| Adjetivo | Comparativo | Superlativo | Traduccion |
|----------|-----------|------------|-----------|
| big | bigger | the biggest | grande |
| fast | faster | the fastest | rapido |
| easy | easier | the easiest | facil |
| light | lighter | the lightest | liviano/claro |
| long | longer | the longest | largo |
| wide | wider | the widest | ancho |
| heavy | heavier | the heaviest | pesado |
| dark | darker | the darkest | oscuro |
| soft | softer | the softest | suave |
| hard | harder | the hardest | duro/dificil |
| short | shorter | the shortest | corto |
| durable | more durable | the most durable | duradero |
| efficient | more efficient | the most efficient | eficiente |
| difficult | more difficult | the most difficult | dificil |
| expensive | more expensive | the most expensive | caro |
| reliable | more reliable | the most reliable | confiable |
| good | **better** | **the best** | bueno |
| bad | **worse** | **the worst** | malo |

### Comparativo de igualdad

- **as + adjetivo + as** → tan ... como
- It's **as fast as** the other server → Es tan rapido como el otro servidor
- The set-up fee is **the same** for both plans → La tarifa es la misma para ambos planes

### Ejemplos del texto modelo (p.15)

- The new layout is **cleaner than** the old one, **so** users **can** find information more easily
- The font is **bigger than** before **because** it is **easier** to read
- This is one of **the best** fonts for product descriptions
- The buttons are **bigger than** the old buttons **so that** users **can click** on them easily
- The new design is **better than** the old design
- This is not **the worst** design choice; it is one of **the best** options

## 8. Conectores (Connectors) (p.16)

Se usan para **justificar decisiones** de diseño. Conectan causa con efecto.

| Conector | Uso | Posicion | Ejemplo |
|----------|-----|---------|---------|
| **because** | Dar una razon | En el medio de la oracion | The font is bigger **because** it is easier to read → La fuente es mas grande porque es mas facil de leer |
| **so** | Mostrar el resultado | En el medio (despues de la causa) | I worked faster **so** I finished on time → Trabaje mas rapido asi que termine a tiempo |
| **so that** | Explicar el proposito | Mas comun, informal y formal | I used CSS **so that** the web page is more responsive → Use CSS para que la pagina sea mas responsive |
| **in order to** | Explicar el proposito | Mas formal, linking word | The designer changed the background **in order to** help users read → El diseñador cambio el fondo para ayudar a leer |

**Diferencia entre "so that" e "in order to":**
- Ambos significan lo mismo (para que / con el fin de)
- **"So that"** es mas comun y se usa en contextos formales e informales
- **"In order to"** es mas formal y menos frecuente

**Ejemplo del Help Box (p.16):**
- We use **"because"** to express a reason: *I created this web page last week **because** the company wanted to publish it today*
- We use **"so"** to show the result: *I worked faster than before **so** I finished on time*

## 9. Pasado Simple (Past Simple Tense) (p.17-18)

Se usa para hablar de **acciones completadas en el pasado** y **decisiones tomadas** sobre el diseño web.

### Estructura

| Forma | Estructura | Ejemplo | Traduccion |
|-------|-----------|---------|-----------|
| **Afirmativa** | Sujeto + verbo en pasado | The designer **changed** the background | El diseñador cambio el fondo |
| **Negativa** | Sujeto + **didn't** + verbo base | He **didn't work** on the project | El no trabajo en el proyecto |
| **Pregunta** | **Did** + sujeto + verbo base? | **Did** she **work** on the CSS? | ¿Ella trabajo en el CSS? |
| **Respuesta corta** | Yes, + sujeto + did / No, + sujeto + didn't | Yes, she **did**. / No, she **didn't**. | Si / No |

**Errores comunes (del booklet):**
- INCORRECTO: He didn't **went** → CORRECTO: He didn't **go**
- INCORRECTO: Did she **worked**? → CORRECTO: Did she **work**?

## 10. Verbo Modal 'Can' (p.18)

Se usa para expresar **posibilidad**, **capacidad** y lo que un sistema o diseño **puede hacer**.

| Forma | Estructura | Ejemplo | Traduccion |
|-------|-----------|---------|-----------|
| **Afirmativa** | Sujeto + **can** + verbo base | Users **can read** the text on different devices | Los usuarios pueden leer el texto en distintos dispositivos |
| **Negativa** | Sujeto + **can't / cannot** + verbo base | You **cannot** change the background | No podes cambiar el fondo |

### Ejemplos del texto modelo (p.15)

- The new layout is cleaner, so users **can find** information more easily
- Users **can read** the text on different devices
- They **can use** the website easily because it is responsive
- Users **can click** on the buttons easily on mobile phones
- I **can't develop** as a web page designer (del texto de Karl, p.19)

## 11. Texto modelo integrador (p.15)

Este texto combina TODOS los temas gramaticales. Es clave para estudiar:

> This web page is for a small online shop. The designer **used** CSS to improve the layout, colours, and fonts. *(past simple)*
> The new layout is **cleaner than** the old one, **so** users **can find** information more easily. *(comparative + connector + can)*
> The content is in the centre **because** this position is **clearer**. *(connector + comparative)*
> The font is **bigger than** before **because** it is **easier** to read. This is one of **the best** fonts for product descriptions. *(comparative + connector + superlative)*
> Last year, the website **had** a dark background and small buttons. Many users **had** problems reading the text. For this reason, the designer **changed** the background colour. *(past simple)*
> Now the background is light **in order to** help users read long texts. *(connector)*
> The buttons are **bigger than** the old buttons **so that** users **can click** on them easily on mobile phones. *(comparative + connector + can)*
> The new design is **better than** the old design. Users **can read** the text on different devices, and they **can use** the website easily **because** it is responsive. *(comparative + can + connector)*
> This is not **the worst** design choice; it is one of **the best** options for a simple business website. *(superlative)*

## 14. Frases utiles para la presentacion oral (Speaking, p.26-27)

### Introducir la pagina
- "This web page is for..." → Esta pagina web es para...
- "This is a simple website for..." → Este es un sitio simple para...
- "I use HTML for structure and CSS for design." → Uso HTML para estructura y CSS para diseño

### Hablar del layout
- "The layout is simple and clean." → El diseño es simple y limpio
- "The content is in the centre because..." → El contenido esta en el centro porque...
- "This layout is **easier** to use **than** the old one." → Este diseño es mas facil de usar que el viejo

### Hablar de la fuente
- "The body font is **bigger than** the menu font." → La fuente del body es mas grande que la del menu
- "This font is **clearer** for users." → Esta fuente es mas clara para los usuarios
- "This is **the best** font for reading." → Esta es la mejor fuente para leer

### Hablar de colores y fondo
- "The background colour is light." → El color de fondo es claro
- "A light background is **better than** a dark background **because**..." → Un fondo claro es mejor que uno oscuro porque...
- "This colour is **the easiest** to read." → Este color es el mas facil de leer

### Hablar de botones
- "The buttons are big and clear." → Los botones son grandes y claros
- "These buttons are **easier** to click on mobile phones." → Estos botones son mas faciles de clickear en celulares
- "This is **the best** option for users." → Esta es la mejor opcion para los usuarios

### Comparar diseños
- "This design is **better than** the old design." → Este diseño es mejor que el viejo
- "The old design was **worse** for mobile users." → El viejo diseño era peor para usuarios mobile

### Expresiones de inicio/cierre para la presentacion
- "In my opinion..." → En mi opinion...
- "One advantage is..." → Una ventaja es...
- "In conclusion..." → En conclusion...

## 15. Checklist para Writing y Speaking

**Contenido:**
- [ ] Describo que es CSS

- [ ] Mis ideas son claras y relevantes al contexto tecnico

- [ ] Uso **can** para hablar de posibilidades

- [ ] Mi parrafo tiene inicio, desarrollo y cierre claros
