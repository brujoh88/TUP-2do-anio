# Cuestionario - Actividad 2: Funciones, POO, DOM y almacenamiento

Resultado: **10/10** (100%)

## Resumen rápido

| Concepto | Clave |
|----------|-------|
| Arrow Function (1 línea) | El `return` es implícito, no es necesario escribirlo |
| Hoisting | Solo aplica a Function Declaration |
| Clase en POO | El "plano" o molde que define propiedades y comportamientos |
| Escuchar clic | `element.addEventListener('click', ...)` |
| Guardar objeto en LocalStorage | Usar `JSON.stringify()` para convertirlo primero |

---

## Preguntas y respuestas

**1. En las Funciones Flecha (Arrow Functions), si la función tiene una sola línea de código, ¿qué sucede con la sentencia `return`?**
- **Respuesta: Es implícita, no es necesario escribirla**

**2. ¿Cuál de los siguientes tipos de declaración de función permite ser llamada antes de ser definida en el código (tiene Hoisting)?**
- **Respuesta: Declaración de Función (Function Declaration)**

**3. En el contexto de la POO en JavaScript, ¿qué es una 'Clase'?**
- **Respuesta: El "plano" o molde que define propiedades y comportamientos**

**4. ¿Cuál es el método correcto para "escuchar" un clic en un elemento HTML desde JavaScript?**
- **Respuesta: `element.addEventListener('click', ...)`**

**5. Para guardar un objeto de JavaScript dentro del LocalStorage, ¿qué método debemos usar para convertirlo primero?**
- **Respuesta: `JSON.stringify()`**
